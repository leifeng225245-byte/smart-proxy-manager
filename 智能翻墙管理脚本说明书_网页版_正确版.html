📋 完整功能说明

🔧 核心命令

命令 功能 说明
on 开启翻墙 启动服务，设置代理，配置浏览器
off 关闭翻墙 停止服务，清除代理，恢复直连
status 查看状态 显示完整的服务和配置状态
help 显示帮助 显示所有可用命令
backup 手动备份 备份当前配置和状态
restore 恢复配置 从备份恢复配置
test 测试网络 测试直连和代理连接

🎯 详细功能介绍

1. 开启翻墙 (on)

./mihomo-manager.sh on

执行流程：

  1. ✅ 备份当前配置
  2. ✅ 启动mihomo服务
  3. ✅ 启用bashrc代理配置
  4. ✅ 设置当前会话环境变量
  5. ✅ 设置GNOME代理为手动模式
  6. ✅ 保存状态为enabled
  7. ✅ 验证网络连通性

输出示例：

开启翻墙... ✅ 配置已备份 启动mihomo服务... ✅ mihomo服务已启动 [2026-03-15 04:33:27] GNOME代理已设置为手动模式 🎉 翻墙已成功开启！

2. 关闭翻墙 (off)

./mihomo-manager.sh off

执行流程：

  1. ✅ 备份当前配置
  2. ✅ 停止mihomo服务
  3. ✅ 注释bashrc代理配置
  4. ✅ 清除当前会话环境变量
  5. ✅ 设置GNOME代理为无代理模式
  6. ✅ 保存状态为disabled
  7. ✅ 验证直连网络

输出示例：

关闭翻墙... ✅ 配置已备份 停止mihomo服务... ✅ mihomo服务已停止 [2026-03-15 04:32:41] GNOME代理已设置为无代理模式 [2026-03-15 04:32:41] 代理配置已禁用 🎉 翻墙已关闭，直连网络正常！

3. 查看状态 (status)

./mihomo-manager.sh status

显示信息：

  • mihomo服务状态（运行中/已停止）
  • 环境变量状态（代理模式/直连模式）
  • GNOME代理状态（手动模式/无代理模式）
  • 端口监听状态（监听中/未监听）
  • 保存状态（enabled/disabled）

输出示例：

当前状态: mihomo服务: 运行中 环境变量: 代理模式 HTTP_PROXY=http://127.0.0.1:7890 HTTPS_PROXY=http://127.0.0.1:7890 GNOME代理: 手动模式 7890端口: 监听中 保存状态: enabled

🎯 状态详解

服务状态

  • 运行中 ✅ - mihomo代理服务正在运行
  • 已停止 ❌ - mihomo代理服务已停止

环境变量状态

  • 代理模式 ✅ - HTTP_PROXY等变量已设置
  • 直连模式 ⚠️ - 代理变量已清除

GNOME代理状态

  • 手动模式 ✅ - 浏览器使用系统代理
  • 无代理模式 ⚠️ - 浏览器直连网络
  • 自动模式 🔵 - 使用自动代理配置
  • 未知模式 ❌ - 代理设置异常

端口监听状态

  • 监听中 ✅ - 7890端口正在监听
  • 未监听 ❌ - 7890端口未监听

保存状态

  • enabled ✅ - 上次保存为开启状态
  • disabled ⚠️ - 上次保存为关闭状态
  • 未设置 ❌ - 没有状态文件

🔍 故障排除

常见问题1：翻墙开启后浏览器无法访问国外网站

症状：

  • 脚本显示开启成功
  • mihomo服务运行中
  • 但浏览器无法访问Google等

解决步骤：

# 1. 检查状态 ./mihomo-manager.sh status # 2. 确认GNOME代理模式 gsettings get org.gnome.system.proxy mode # 3. 如果不是'manual'，重新开启 ./mihomo-manager.sh off ./mihomo-manager.sh on

常见问题2：翻墙关闭后浏览器无法访问国内网站

症状：

  • 脚本显示关闭成功
  • mihomo服务已停止
  • 但浏览器无法访问百度等

解决步骤：

# 1. 检查状态 ./mihomo-manager.sh status # 2. 确认GNOME代理模式 gsettings get org.gnome.system.proxy mode # 3. 如果不是'none'，重新关闭 ./mihomo-manager.sh on ./mihomo-manager.sh off

常见问题3：脚本执行失败

症状：

  • 命令执行报错
  • 权限不足
  • 服务启动失败

解决步骤：

# 1. 检查脚本权限 ls -la ~/mihomo-manager.sh # 2. 添加执行权限 chmod +x ~/mihomo-manager.sh # 3. 检查mihomo服务 systemctl status mihomo

常见问题4：环境变量不生效

症状：

  • 脚本执行成功
  • 但环境变量没有设置

解决步骤：

# 1. 手动设置环境变量 export HTTP_PROXY=http://127.0.0.1:7890 export HTTPS_PROXY=http://127.0.0.1:7890 export ALL_PROXY=http://127.0.0.1:7890 # 2. 重新加载bashrc source ~/.bashrc # 3. 验证设置 env | grep -i proxy
