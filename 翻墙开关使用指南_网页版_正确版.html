🔧 故障排除

问题1：翻墙开启后浏览器无法访问国外网站

# 检查状态 ./mihomo-manager.sh status # 确认GNOME代理是否为"手动模式" gsettings get org.gnome.system.proxy mode # 如果不是手动模式，重新开启 ./mihomo-manager.sh off ./mihomo-manager.sh on

问题2：翻墙关闭后浏览器无法访问国内网站

# 检查状态 ./mihomo-manager.sh status # 确认GNOME代理是否为"无代理模式" gsettings get org.gnome.system.proxy mode # 如果不是无代理模式，重新关闭 ./mihomo-manager.sh on ./mihomo-manager.sh off

问题3：脚本执行失败

# 检查脚本权限 ls -la ~/mihomo-manager.sh # 如果没有执行权限，添加权限 chmod +x ~/mihomo-manager.sh # 检查mihomo服务状态 systemctl status mihomo

🎊 使用技巧

⚡ 快速切换

# 创建别名（可选） alias proxy-on='./mihomo-manager.sh on' alias proxy-off='./mihomo-manager.sh off' alias proxy-status='./mihomo-manager.sh status' # 使用别名 proxy-on # 开启翻墙 proxy-off # 关闭翻墙 proxy-status # 查看状态

📋 状态检查脚本

# 创建状态检查脚本 cat > check-proxy.sh << 'EOF' #!/bin/bash echo "=== 翻墙状态检查 ===" ./mihomo-manager.sh status echo "" echo "=== 网络测试 ===" echo "测试百度：" curl -I --connect-timeout 3 http://www.baidu.com 2>/dev/null | head -1 echo "测试Google：" curl -I --connect-timeout 3 --proxy http://127.0.0.1:7890 https://www.google.com 2>/dev/null | head -1 EOF chmod +x check-proxy.sh ./check-proxy.sh
