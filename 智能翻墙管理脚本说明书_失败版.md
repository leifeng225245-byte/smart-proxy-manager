## 📖 完整功能说明

### 🎮 核心命令

| 命令 | 功能 | 使用场景 |
|------|------|----------|
| `on` | 开启翻墙 | 需要访问外网时 |
| `off` | 关闭翻墙 | 需要直连国内网络时 |
| `status` | 查看状态 | 检查当前翻墙状态 |
| `test` | 测试网络 | 验证网络连通性 |
| `backup` | 手动备份 | 重要操作前备份 |
| `help` | 显示帮助 | 查看所有命令 |

## 🛠️ 高级功能

### 📦 自动备份
脚本每次操作前都会自动备份配置文件：
- ~/.bashrc
- ~/.config/mihomo/config.yaml
- 当前状态信息
- 环境变量设置

**备份位置：**
```
/media/lsk/future/proxy-config-backup/smart-backup/configs/
```

**备份命名：**
```
backup_YYYYMMDD_HHMMSS/
├── bashrc.bak
├── config.yaml.bak
├── status.txt
└── env_vars.txt
```

### 📝 日志记录
所有操作都会记录到日志文件：
```
/media/lsk/future/proxy-config-backup/smart-backup/logs/mihomo-manager.log
```

**日志格式：**
```
[2026-03-15 03:58:01] 翻墙已开启
[2026-03-15 03:56:43] 翻墙已关闭，直连正常
[2026-03-15 03:56:33] 配置已备份到: backup_20260315_035633
```

### 🧠 状态记忆
脚本会记住你的选择：
- **enabled**: 重启后自动开启翻墙
- **disabled**: 重启后保持关闭状态

**状态文件：**
```

```

## ⚠️ 注意事项

### 🔒 安全提醒
- 脚本会修改系统配置文件
- 每次操作前自动备份
- 有问题可以恢复备份

### 🐛 故障排除

#### 问题1：命令执行失败
```bash
# 检查脚本权限
chmod +x ~/mihomo-manager.sh

# 检查脚本是否存在
ls -la ~/mihomo-manager.sh
```

#### 问题2：翻墙开启但无法访问
```bash
# 检查服务状态
sudo systemctl status mihomo

# 检查端口监听
ss -tlnp | grep 7890

# 检查环境变量
env | grep proxy
```

#### 问题3：关闭后仍无法直连
```bash
# 手动清除环境变量
unset HTTP_PROXY HTTPS_PROXY ALL_PROXY

# 检查bashrc配置
grep -n "export.*PROXY" ~/.bashrc

# 重启网络服务
sudo systemctl restart NetworkManager
```

#### 问题4：状态显示不正确
```bash
# 重新设置环境变量
source ~/.bashrc

# 检查状态文件
cat ~/.mihomo_status
```
