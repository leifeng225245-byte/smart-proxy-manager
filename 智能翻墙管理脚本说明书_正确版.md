## 🔍 故障排除

### 常见问题1：翻墙开启后浏览器无法访问国外网站

**症状：**
- 脚本显示开启成功
- mihomo服务运行中
- 但浏览器无法访问Google等

**解决步骤：**
```bash
# 1. 检查状态
./mihomo-manager.sh status

# 2. 确认GNOME代理模式
gsettings get org.gnome.system.proxy mode

# 3. 如果不是'manual'，重新开启
./mihomo-manager.sh off
./mihomo-manager.sh on

# 4. 检查环境变量
env | grep -i proxy
```

### 常见问题2：翻墙关闭后浏览器无法访问国内网站

**症状：**
- 脚本显示关闭成功
- mihomo服务已停止
- 但浏览器无法访问百度等

**解决步骤：**
```bash
# 1. 检查状态
./mihomo-manager.sh status

# 2. 确认GNOME代理模式
gsettings get org.gnome.system.proxy mode

# 3. 如果不是'none'，重新关闭
./mihomo-manager.sh on
./mihomo-manager.sh off

# 4. 清除浏览器缓存
# 重启浏览器
```

### 常见问题3：脚本执行失败

**症状：**
- 命令执行报错
- 权限不足
- 服务启动失败

**解决步骤：**
```bash
# 1. 检查脚本权限
ls -la ~/mihomo-manager.sh

# 2. 添加执行权限
chmod +x ~/mihomo-manager.sh

# 3. 检查mihomo服务
systemctl status mihomo

# 4. 检查配置文件
ls -la ~/.config/mihomo/config.yaml
```

### 常见问题4：环境变量不生效

**症状：**
- 脚本执行成功
- 但环境变量没有设置

**解决步骤：**
```bash
# 1. 手动设置环境变量
export HTTP_PROXY=http://127.0.0.1:7890
export HTTPS_PROXY=http://127.0.0.1:7890
export ALL_PROXY=http://127.0.0.1:7890

# 2. 重新加载bashrc
source ~/.bashrc

# 3. 验证设置
env | grep -i proxy
```
